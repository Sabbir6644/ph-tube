Scope of var, let, and const:


var:


Variables declared with var are function-scoped or globally scoped, depending on where they are declared. If declared within a function, the variable is accessible only within that function. If declared outside any function, the variable becomes a global variable and is accessible throughout the entire script. var declarations are hoisted to the top of their containing function or script, which can lead to unexpected behavior.


let:


Variables declared with 'let' are block-scoped, which means they are only accessible within the block (a pair of curly braces) where they are defined. 'let' variables are not hoisted to the top of their scope and must be declared before they are used.


const:


Variables declared with const are also block-scoped, just like let. The key difference is that once a value is assigned to a const variable, it cannot be reassigned. It is constant. However, the value itself is not immutable. If the value is an object, the properties of the object can still be changed.

Use cases of null and undefined


In JavaScript, both `null` and `undefined` are used to represent the absence of a value. However, they are used in slightly different contexts:


Undefined:

It indicates that a variable has been declared but has not been assigned a value yet. It is also the default value of function parameters that are not provided when the function is called.


There are some scenarios where we get undefined:

  • A variable that has been declared but has not been assigned a value is undefined.
  • Function parameters that are not provided are undefined.
  • When you access an object property that doesn't exist, the result is undefined.
  • The return value of a function that doesn't explicitly return anything is undefined.
  • Array Elements That Don't Exist
  • Implicit Return in Arrow Functions: If you have an arrow function with an implicit return (no braces {}), and it doesn't have a value to return, it will return undefined.
  • Asynchronous Code Without Proper Handling: When working with asynchronous code, if you don't handle promises or callbacks correctly, variables may not be defined when you expect them to be.

null:


It is a value that represents the intentional absence of any object value. It is often used by developers to indicate that a variable or object property has no value or is empty.


REST API:


REST API stands for Representational State Transfer Application Programming Interface. It is a set of architectural principles and constraints for designing networked applications. REST is not a technology itself; it's a style of architecture that uses HTTP requests to perform CRUD (Create, Read, Update, Delete) operations on resources.
